# SOURCE MODULE

## MODULE ID
UYGUNAYAKKABI-STOREFRONT-RUNTIME-HISTORY-V2

## Domain Category
Frontend / Runtime Stability

## Core Purpose
Document storefront runtime issues and their resolutions for future reference.

## Resolved Issues (History)

### Issue: ReferenceError — Cannot access 'U' before initialization
- **Status**: RESOLVED
- **Cause**: Helper function `U` was declared after it was used by `heroImg`
- **Fix**: Reordered function declarations in UygunApp.jsx
- **Commit**: Part of earlier stabilization work

### Issue: next/image remotePatterns blocking product images
- **Status**: RESOLVED
- **Cause**: `next/image` enforces remotePatterns validation; admin-uploaded images and Unsplash URLs were blocked
- **Fix**: Replaced `next/image` with plain `<img>` tags in ProductImages.tsx and UygunApp.jsx (D-025)
- **Commit**: 8c3b520

### Issue: Media URLs not resolving correctly
- **Status**: RESOLVED
- **Cause**: `getMediaUrl()` was constructing `/media/${filename}` but Payload stores a `url` field on media documents
- **Fix**: Updated to check `first.url` first, fallback to `/media/${first.filename}`; added `staticURL: '/media'` to Media.ts
- **Commit**: 8c3b520

### Issue: Empty product images
- **Status**: RESOLVED
- **Cause**: No real images in static product data
- **Fix**: Added 5 Unsplash shoe photos + 3 wallet photos rotated across all 39 products
- **Commit**: 30f6825

## Current State (2026-03-09)
- Storefront is functionally complete with ~817 lines in UygunApp.jsx
- 39 static products across 8 categories with real images
- Dynamic content from SiteSettings and Banners (server-side fetched)
- All WhatsApp links, trust badges, promo banners driven by admin settings
- Inline-style token system (no Tailwind dependency)
- Awaiting user runtime validation
