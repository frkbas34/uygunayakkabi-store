Routes incoming Telegram intent to the correct specialist skill.

Formal JSON schema with confidence × risk gate for every action.

12-layer memory: identity, policies, runbooks, incidents, traces, patterns, decisions, evaluations, rewards, evals, summaries, archive.

6 Diagnostic Entry Points
🔴 A — Product Not Visible
1. products.status == 'active'?
2. Storefront query intact?
3. ENABLE_STATIC_FALLBACK = false?
4. force-dynamic on page.tsx?
5. Vercel deployment current?
6. price > 0 (activation gate)?
7. Neon DB reachable?
🟡 B — Wrong Product Data
1. parseConfidence < 70?
2. parseWarnings[]?
3. automationMeta.rawCaption?
4. Explicit fields override parsed?
5. autoDecisionReason?
6. DB record directly?
🔵 C — Image Not Rendering
1. products.images[] populated?
2. media.product reverse lookup?
3. Blob URL absolute HTTPS?
4. extractMediaUrls() output?
5. NEXT_PUBLIC_SERVER_URL set?
6. BLOB_READ_WRITE_TOKEN valid?
🔘 D — Channel Dispatch Failed
1. channelTargets includes channel?
2. AutomationSettings.publishX?
3. channels.publishX not false?
4. N8N_CHANNEL_X_WEBHOOK set?
5. n8n workflow active?
6. dispatchNotes in sourceMeta?
🟢 E — Telegram Intake Failed
1. Bot online? (uptime-kuma)
2. Group in allowlist? requireMention?
3. n8n webhook active (not test)?
4. AUTOMATION_SECRET match?
5. Duplicate detected?
6. Media attach step succeeded?
🔳 F — Stock / Price Mismatch
1. DB: actual price / stock_quantity?
2. Variant records linked?
3. Telegram STOCK command used?
4. InventoryLogs entries?
5. Caption price parsing correct?
6. External sync overwrote value?

Outcome / Evaluation / Reward — Strict Separation
OUTCOME
What actually happened in the system?
“Product became visible after status changed to active”
stored in traces/
EVALUATION
Was Mentix's diagnosis correct?
“Agent found correct root cause: status was draft, not a DB issue”
stored in evaluations/
REWARD
Score from outcome + evaluation
“+5 (correct root cause, human confirmed, first-pass)”
stored in rewards/
Reward Framework

✓ Positive Signals

Correct root cause found+5
Issue prevented proactively+5
Human confirmed diagnosis+4
First-pass resolution+3
Recommendation reused later+3
Accurate prediction+3
Minimal diagnostic steps+2

✗ Negative Signals

Wrong diagnosis-5
Risky action without evidence-5
Repeated wrong suggestion-4
High confidence, wrong answer-4
Symptom confused with root cause-3
Missed obvious diagnostic path-3
Unnecessary tool usage-2
